Complete Installation Guide
Server Environment Dependencies

Install these on the machine running the StreamSwarm server/dashboard

Linux Server (Ubuntu/Debian)
# Update package repositories
sudo apt update

# Python 3.9+ and development tools
sudo apt install python3.9 python3.9-pip python3.9-venv python3.9-dev
sudo apt install build-essential libffi-dev libssl-dev

# Database support (PostgreSQL recommended)
sudo apt install postgresql postgresql-contrib libpq-dev

# AI/ML dependencies (server-side analytics)
sudo apt install python3-numpy python3-scipy python3-sklearn
sudo apt install libatlas-base-dev liblapack-dev gfortran

# Web server dependencies
sudo apt install nginx

# Verify installation
python3.9 --version
pip3.9 --version
Note: Replace python3.9 with python3.10 or python3.11 for newer Ubuntu versions
Linux Server (CentOS/RHEL/Fedora)
# Install Python and development tools
sudo dnf install python3.9 python3.9-pip python3.9-devel
sudo dnf install gcc openssl-devel libffi-devel

# Database support (PostgreSQL)
sudo dnf install postgresql-server postgresql-contrib postgresql-devel

# AI/ML dependencies (server-side analytics)
sudo dnf install python3-numpy python3-scipy python3-scikit-learn
sudo dnf install atlas-devel lapack-devel gcc-gfortran

# Web server dependencies
sudo dnf install nginx

# Initialize PostgreSQL (first time only)
sudo postgresql-setup --initdb
sudo systemctl enable postgresql
sudo systemctl start postgresql
Older systems: Replace 'dnf' with 'yum' for CentOS 7/RHEL 7

Firewall Requirements & External Dependencies
Important for Corporate Environments

StreamSwarm requires specific network access for various features. Configure firewalls and proxy settings according to these requirements.

Feature-Specific Network Requirements
Feature External Access Required Ports/Protocols Impact if Blocked
Core Monitoring StreamSwarm Server HTTP/HTTPS (80/443) Complete loss of functionality
Bandwidth Testing Speed test servers HTTP/HTTPS (80/443) No bandwidth measurements
Geolocation Maps ip-api.com HTTP (80) No network path visualization
DNS Tests DNS servers UDP (53) Hostname-based tests fail
System Monitoring None Local only Always works offline
AI/ML Diagnostics None Local only Zero-trust architecture
Server-Side External Dependencies
Required Internet Access
  • Geolocation Services: ip-api.com (HTTP port 80)
  • Purpose: IP geolocation lookup for network path visualization
  • Impact: Without access, geolocation maps will not generate
  • Alternative: Disable geolocation features for air-gapped environments

Client-Side External Dependencies
Required Internet Access
  • StreamSwarm Server: HTTP/HTTPS connection to your server
  • Ports: 80/443 (default) or custom port configured
  • Impact: Core functionality requires server connectivity
Optional Internet Access
  • Speed Test Servers: Various HTTP/HTTPS endpoints
  • Examples: speedtest.net, fast.com, Google speed test servers
  • DNS Resolution: UDP port 53 to DNS servers
  • Purpose: Hostname resolution for network targets

Firewall Configuration Examples
Server Firewall (iptables)
# Allow inbound connections to StreamSwarm server
iptables -A INPUT -p tcp --dport 5000 -j ACCEPT

# Allow outbound for geolocation services
iptables -A OUTPUT -p tcp --dport 80 -d ip-api.com -j ACCEPT

# Allow outbound DNS
iptables -A OUTPUT -p udp --dport 53 -j ACCEPT
Corporate Firewall Rules
# Required: Client to Server communication
ALLOW TCP 80/443 FROM client_subnets TO streamswarm_server

# Optional: Speed test access (if bandwidth testing needed)
ALLOW TCP 80/443 FROM client_subnets TO speedtest_servers

# Optional: Geolocation services (server-side)
ALLOW TCP 80 FROM streamswarm_server TO ip-api.com

# Required: DNS resolution
ALLOW UDP 53 FROM any TO dns_servers
Air-Gapped/Isolated Environments
Fully Offline Capabilities
  • System resource monitoring (CPU, memory, disk)
  • Local network interface statistics
  • Ping tests to internal IP addresses
  • Basic network connectivity tests
  • AI/ML diagnostics (local processing only)
Recommendations for Restricted Environments
  1. Disable geolocation features if ip-api.com access unavailable
  2. Use IP addresses instead of hostnames for network targets
  3. Configure internal speed test servers for bandwidth testing
  4. Use SQLite database to avoid external database dependencies
Database Setup & Configuration
PostgreSQL Setup

StreamSwarm uses PostgreSQL for data storage. Here's how to set it up:

Install PostgreSQL:
# Ubuntu/Debian
sudo apt install postgresql postgresql-contrib

# CentOS/RHEL
sudo yum install postgresql-server postgresql-contrib
sudo postgresql-setup initdb
sudo systemctl enable postgresql
sudo systemctl start postgresql
Create Database:
sudo -u postgres psql

CREATE DATABASE streamswarm;
CREATE USER streamswarm_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE streamswarm TO streamswarm_user;
\q
Configuration Options

StreamSwarm uses environment variables for configuration. You have three options:

Option 1: Environment Variables (Recommended)
# Set in your shell (Linux/macOS)
export DATABASE_URL="postgresql://streamswarm_user:your_password@localhost:5432/streamswarm"
export SESSION_SECRET="your-secret-key-here"

# Set in PowerShell (Windows)
$env:DATABASE_URL="postgresql://streamswarm_user:your_password@localhost:5432/streamswarm"
$env:SESSION_SECRET="your-secret-key-here"
Option 2: .env File (Development)
# Create .env file in project root
echo "DATABASE_URL=postgresql://streamswarm_user:your_password@localhost:5432/streamswarm" > .env
echo "SESSION_SECRET=your-secret-key-here" >> .env

# The application will automatically load these variables
Option 3: Modify config.py (Not Recommended)
# In config.py, change the default values:
DATABASE_URL = os.getenv('DATABASE_URL', 'your-database-url-here')
SESSION_SECRET = os.getenv('SESSION_SECRET', 'your-secret-key-here')

# Warning: Never commit real credentials to version control!
About SESSION_SECRET

Purpose: Encrypts user sessions and cookies for web security

Requirements:

  • Minimum 32 characters long
  • Random mix of letters, numbers, and symbols
  • Unique for each deployment
  • Never shared or committed to version control

Generate one: openssl rand -hex 32 or use online generator

Security Note: Use a strong, unique password and session secret in production. Never commit credentials to version control. The SESSION_SECRET protects user sessions from tampering and hijacking.
How Configuration Works

StreamSwarm reads configuration in this priority order:

  1. Environment variables (highest priority)
  2. .env file (if no environment variable set)
  3. Default values in config.py (fallback)

The application automatically uses SQLite for development if no DATABASE_URL is provided. All other settings have sensible defaults in config.py.

Test Database Connection:
psql "$DATABASE_URL" -c "SELECT version();"

Understanding the Architecture
Server Component
  • Central control hub for all monitoring
  • Hosts web dashboard for visualization
  • Stores data in PostgreSQL database
  • Schedules tests across multiple clients
  • Provides REST API access
When to deploy a server:
You need centralized monitoring, test coordination, historical data storage, or management interfaces.
Client Component
  • Lightweight monitoring agent
  • Executes network tests (ping, traceroute)
  • Collects system metrics (CPU, memory, disk)
  • Reports data to server in real-time
  • Receives test assignments from server
When to deploy a client:
You want to monitor from a specific location, test connectivity to services, or participate in distributed testing.

Service Configuration

Run StreamSwarm as a system service:

Systemd Service (Linux):
# Create /etc/systemd/system/streamswarm.service
[Unit]
Description=StreamSwarm Network Monitor
After=network.target

[Service]
Type=exec
User=streamswarm
WorkingDirectory=/opt/streamswarm
Environment=DATABASE_URL=postgresql://...
Environment=SESSION_SECRET=your-secret-here
ExecStart=/opt/streamswarm/venv/bin/gunicorn --bind 0.0.0.0:5000 main:app
Restart=always

[Install]
WantedBy=multi-user.target

Metrics Definition Guide

Comprehensive reference for all performance metrics collected during network monitoring tests. Each metric provides specific insights into system and network behavior.

Network Performance
Network Latency (ms)

Round-trip time for ICMP ping packets to destination.

<50ms: Excellent 50-150ms: Good >150ms: Poor
Packet Loss (%)

Percentage of packets lost during transmission.

0%: Perfect 1-5%: Acceptable >5%: Critical
Network Jitter (ms)

Variation in packet arrival times indicating network stability.

<5ms: Stable 5-20ms: Moderate >20ms: Unstable
Traceroute Hops

Number of network devices between client and destination.

Shows network path complexity
Bandwidth Upload/Download (Mbps)

Data transfer speeds for upload and download operations.

Measures connection capacity
Advanced Network Analysis
DNS Resolution Time (ms)

Time to resolve domain names to IP addresses.

<50ms: Fast 50-200ms: Normal >200ms: Slow
TCP Connect Time (ms)

Time to establish TCP connection handshake.

Measures connection establishment speed
SSL Handshake Time (ms)

Time to complete SSL/TLS security negotiation.

Indicates encryption overhead
Time to First Byte (TTFB) (ms)

Time from request sent to first response byte received.

<200ms: Fast 200-1000ms: Normal >1000ms: Slow
System Performance
CPU Usage (%)

Percentage of CPU processing power currently in use.

<70%: Normal 70-90%: High >90%: Critical
Memory Usage (%)

Percentage of system RAM currently allocated.

<80%: Good 80-95%: High >95%: Critical
Disk Usage (%)

Percentage of storage space currently used.

<80%: Safe 80-95%: Monitor >95%: Critical
CPU Load Average

System load over 1, 5, and 15 minute intervals.

Values relative to CPU core count
Process Count

Total number of running processes on the system.

50-150: Light 150-300: Normal >300: Heavy
Memory & Storage Details
Memory Available (GB)

Amount of RAM available for new processes.

Free memory for applications
Memory Cached (GB)

RAM used for file system caching to improve performance.

System optimization
Memory Buffers (GB)

RAM used for I/O buffers and kernel operations.

Kernel memory management
Swap Usage (GB/%)

Virtual memory usage when RAM is full.

0%: Optimal 1-10%: Monitor >10%: Problem
Disk Read/Write IOPS

Input/Output operations per second for storage devices.

Storage performance indicator
Network Interface Detection
Primary Interface Type

Automatic detection of connection type for troubleshooting context.

Ethernet: Stable Wireless: Variable VPN: Tunneled
Connection Speed (Mbps/Gbps)

Reported link speed capacity of the network interface.

1-10 Gbps: High-speed 100 Mbps: Standard 10 Mbps: Legacy
Duplex Mode

Communication mode affecting throughput potential.

Full: Optimal Half: Limited
Wireless SSID & Signal

WiFi network name and signal strength for wireless clients.

Network Interface Troubleshooting

Interface Type Correlation: Wireless clients typically show higher latency and jitter than ethernet. VPN connections add encryption overhead.

Speed Limitations: Performance cannot exceed interface speed. 100 Mbps ethernet limits bandwidth regardless of ISP capacity.

Wireless Analysis: Signal strength below -70 dBm often causes packet loss and latency spikes. Check frequency band and interference.

Duplex Issues: Half-duplex connections cause collisions and reduced throughput. Full-duplex is essential for modern networks.

Understanding Metric Relationships

Network Issues: High latency with packet loss often indicates congestion. Jitter spikes suggest unstable routing.

System Performance: High CPU with low memory usage suggests compute-bound tasks. High memory with normal CPU indicates memory-intensive applications.

Correlation Analysis: Network performance degrades when system resources (CPU/Memory) exceed 90% usage. Monitor both categories together.

GNMI Managed Infrastructure Analysis
What is GNMI?

GNMI (gRPC Network Management Interface) enables direct telemetry collection from enterprise network equipment for hop-by-hop infrastructure analysis.

Enterprise Feature

GNMI analysis provides detailed performance insights for managed network infrastructure, complementing standard traceroute data with device-level metrics.

Telemetry Data Collected
  • Processing Latency: Packet forwarding delays within devices
  • Queue Analysis: Buffer utilization and queue depth statistics
  • CPU Utilization: Device processing load and capacity
  • Interface Performance: Bandwidth, error rates, packet statistics
Setup Requirements
Infrastructure Requirements
  • GNMI-Enabled Devices: Modern enterprise routers, switches, firewalls
  • Network Access: Client must reach device management interfaces
  • Authentication: Device credentials configured in client
  • Supported Vendors: Cisco, Juniper, Arista, etc.
Client Configuration

Python Package:

Certificate-Based Authentication (Recommended)

Why Use Certificates?

  • Enhanced Security: No passwords to manage or rotate
  • PKI Integration: Works with enterprise certificate infrastructure
  • Audit Trail: Better logging and compliance capabilities
  • Zero Passwords: Eliminates password-based attack vectors
Step 1: Generate Client Certificates
# Create certificate directory
sudo mkdir -p /etc/streamswarm/certs
cd /etc/streamswarm/certs

# Generate private key (2048-bit RSA)
openssl genrsa -out client.key 2048

# Create certificate signing request
openssl req -new -key client.key -out client.csr \
  -subj "/CN=streamswarm-client/O=YourOrg/C=US"

# Self-signed certificate (365 days)
openssl x509 -req -days 365 -in client.csr \
  -signkey client.key -out client.crt

# Set proper permissions
sudo chmod 600 client.key
sudo chmod 644 client.crt
Enterprise Tip

For production, get certificates signed by your enterprise CA rather than using self-signed certificates.

Step 2: Configure Certificate Authentication
# Pure certificate authentication
self.gnmi_analyzer.add_device(
    device_ip="192.168.1.1",
    port=830,
    auth_method='certificate',
    cert_file='/etc/streamswarm/certs/client.crt',
    key_file='/etc/streamswarm/certs/client.key',
    ca_file='/etc/streamswarm/certs/ca.crt'
)

# Certificate + username (some devices)
self.gnmi_analyzer.add_device(
    device_ip="10.0.1.5",
    username="admin",
    port=32768,
    auth_method='cert_username',
    cert_file='/etc/streamswarm/certs/client.crt',
    key_file='/etc/streamswarm/certs/client.key'
)
